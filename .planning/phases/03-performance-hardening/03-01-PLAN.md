<task type="auto">
  <name>Task 1: Implement Header Injection in Service Worker</name>
  <files>sw.js</files>
  <action>
    Merge the COOP/COEP header injection logic into the existing 'fetch' event listener in sw.js.
    
    Specifics:
    - Only intercept GET requests.
    - For successful fetches (status 200 or cached), create a new Response with headers:
      - 'Cross-Origin-Embedder-Policy': 'require-corp'
      - 'Cross-Origin-Opener-Policy': 'same-origin'
    - Ensure the existing caching logic (vendor cache-first, others network-first) remains intact.
    - Avoid opaque responses (status 0) as they cannot be modified.
  </action>
  <verify>
    <automated>npx playwright test tests/e2e/network.spec.js</automated>
  </verify>
  <done>GET requests in the Network tab show COOP and COEP headers after SW activation.</done>

<task type="auto">
  <name>Task 2: Handle Isolation Reload in UI</name>
  <files>ui/app.js</files>
  <action>
    Add a check at the top of ui/app.js to detect if the environment is Cross-Origin Isolated.
    
    Implementation:
    - If window.crossOriginIsolated is false and navigator.serviceWorker.controller is active:
      - Trigger a location.reload() to apply the newly injected headers.
    - If no controller is active, wait for the service worker to become ready/active before reloading.
    - Rationale: Headers injected by the SW are only visible to the page after it has been loaded UNDER the SW's control.
  </action>
  <verify>
    <automated>npx playwright test tests/e2e/pwa_ui.spec.js</automated>
  </verify>
  <done>Application reloads exactly once when first loaded or when SW is updated, resulting in window.crossOriginIsolated === true.</done>
</task>
